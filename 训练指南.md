# MIRA 模型训练指南

## 1. 环境要求

- Python 3.11+
- PyTorch + CUDA
- HuggingFace Transformers
- 推荐硬件: 单卡 NVIDIA GPU (24GB+ 显存)

## 2. 项目结构

```
/home/zjl/MIRA/
├── main.py                 # 训练入口，解析参数
├── train_loop.sh           # 自动重启脚本（崩溃后自动恢复）
├── mira/
│   ├── runner.py           # 核心训练逻辑
│   ├── trainer/            # 自定义 Trainer + LR 调度器
│   ├── models/             # MIRA 模型定义
│   └── datasets/           # 数据集定义
└── ppg_full/               # 训练数据目录
    └── ppg_data.npy        # memmap 格式数据
```

## 3. 启动训练

### 方式一：使用自动重启脚本（推荐）

自动重启脚本会在训练崩溃时自动恢复，最多重试 10 次，始终从最新 checkpoint 恢复。

```bash
cd /home/zjl/MIRA
bash train_loop.sh
```

### 方式二：直接启动

```bash
cd /home/zjl/MIRA
torchrun --master_addr=localhost --master_port=9899 \
    --node_rank=0 --nproc_per_node=1 --nnodes=1 \
    main.py \
    --model_path Maple728/TimeMoE-50M \
    --data_path ppg_full \
    --output_path ppg_output \
    --max_length 1024 \
    --normalization_method zero \
    --attn_implementation eager \
    --micro_batch_size 52 \
    --global_batch_size 52 \
    --precision bf16 \
    --save_strategy steps \
    --save_steps 100 \
    --gradient_checkpointing \
    --save_only_model \
    --save_total_limit 5 \
    --logging_steps 10 \
    --lr_scheduler_type cosine \
    --warmup_ratio 0.03 \
    --warmup_start_lr 1e-4 \
    --weight_decay 0.1 \
    --num_train_epochs 1.0 \
    --dataloader_num_workers 8
```

## 4. 参数说明

### 模型与数据

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--model_path` | `Maple728/TimeMoE-50M` | 预训练模型路径 |
| `--data_path` | - | 训练数据目录（含 `ppg_data.npy`） |
| `--output_path` | `logs/time_moe` | 模型输出和 checkpoint 保存路径 |
| `--max_length` | `1024` | 序列窗口长度 |
| `--normalization_method` | `zero` | 数据归一化方法 (`none`, `zero`, `max`) |

### 训练参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--precision` | `fp16` | 精度模式 (`fp16`, `bf16`, `fp32`) |
| `--micro_batch_size` | `16` | 每卡 micro batch size |
| `--global_batch_size` | `16` | 全局 batch size |
| `--gradient_accumulation_steps` | 自动计算 | 梯度累积步数（自动计算） |
| `--num_train_epochs` | `1.0` | 训练轮数 |
| `--train_steps` | `None` | 指定训练步数（覆盖 epochs） |
| `--gradient_checkpointing` | 关闭 | 启用梯度检查点以节省显存 |

### 学习率调度

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--learning_rate` | `5e-5` | 基础学习率 |
| `--lr_scheduler_type` | `cosine` | 调度器类型 (`cosine`, `linear`, `constant`) |
| `--warmup_ratio` | `0.03` | warmup 占总步数比例 |
| `--warmup_start_lr` | `1e-4` | warmup 起始 LR（相对于基础 LR 的比值） |
| `--weight_decay` | `0.1` | 权重衰减 |

### 检查点与日志

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--save_strategy` | `no` | 保存策略 (`steps`, `epoch`, `no`) |
| `--save_steps` | `None` | 每 N 步保存一次 |
| `--save_total_limit` | `None` | 最多保留的 checkpoint 数量 |
| `--save_only_model` | 开启 | 只保存模型权重（不保存优化器状态，节省磁盘） |
| `--logging_steps` | `1` | 每 N 步打印一次 loss |
| `--from_scratch` | 关闭 | 从头训练（不加载预训练权重） |

## 5. GPU 调优建议

### 显存优化

- 开启 `--gradient_checkpointing`：减少约 60% 显存占用，速度略有下降
- 推荐配置：`--micro_batch_size 52`（单卡 24GB 显存）
- 显存占用约 20GB/24GB（80%+），GPU 利用率 100%

### 如果遇到 OOM（显存不足）

1. 减小 `--micro_batch_size`（从 52 降到 32 或更低）
2. 确保开启 `--gradient_checkpointing`
3. 使用 `bf16` 而非 `fp32`

### 数据加载加速

- `--dataloader_num_workers 8`：增加 DataLoader worker 数量

## 6. 训练恢复

### 从 checkpoint 恢复

训练脚本会自动检测 `output_path` 下的最新 checkpoint 并恢复：

```bash
# train_loop.sh 会自动检测并恢复
bash train_loop.sh
```

Checkpoint 格式：`ppg_output/checkpoint-5000/`

### 停止训练

```bash
# 找到并杀死所有训练进程
kill $(ps aux | grep "main.py" | grep -v grep | awk '{print $2}')
```

## 7. 常见问题

### 端口被占用

```bash
# 释放端口
kill $(lsof -t -i:9899)
```

### 预训练模型下载失败

模型会加载到本地缓存。确保网络可访问 HuggingFace，或预先下载：

```bash
export HF_HUB_OFFLINE=1  # 离线模式
```

### 训练速度异常慢

Checkpoint 恢复阶段会重建数据迭代器，速度较慢（约 5 it/s），这是正常现象。越过 checkpoint 后速度会恢复正常（约 7-8 秒/步）。

### 查看训练进度

```bash
# 查看 GPU 状态
nvidia-smi

# 查看最新 checkpoint
ls -lt ppg_output/checkpoint-*

# 查看 TensorBoard
tensorboard --logdir ppg_output/tb_logs
```

## 8. 模型信息

- **参数量**: 113.428M
- **架构**: Mixture-of-Experts (MoE)
- **专家数**: 8（每次激活 2 个）
- **层数**: 12 Transformer 层
- **隐藏层**: 384
- **注意力头数**: 12
- **位置编码**: CT-RoPE（连续时间旋转位置编码）
- **解算器**: dopri5 ODE 求解器
